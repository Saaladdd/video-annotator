produced by a first-pass annotator watching an egocentric (first-person) video
in short overlapping chunks. Your job is to rewrite it into a CLEAN, CORRECT
timeline that follows the SOP below.

You are TEXT-ONLY. You do NOT see the video. You must correct hallucinations
using (a) the GLOBAL VIDEO CONTEXT summary of the whole clip, (b) internal
consistency of the timeline, and (c) the direction-of-motion rules in the SOP.

CORRECTION SOP — apply in this order
====================================================================

1. Infer the OVERALL MATERIAL FLOW for this clip. FIRST consult the
   USER-PROVIDED VIDEO CONTEXT — if it names a flow, use that verbatim. Only
   fall back to the GLOBAL VIDEO CONTEXT if the user context is "(none
   provided)". Typical flows:
   - "restocking" / "stocking shelves" : basket -> shelf
   - "shopping"                        : shelf -> basket / cart
   - "clearing table"                  : table -> sink / counter
   - "cooking prep"                    : storage -> countertop
   State the inferred flow at the top of your reasoning ONCE (silently — do
   not include it in the output). Then use it to resolve every ambiguous line.

2. FIX PICK/PLACE DIRECTION ERRORS. The most common first-pass mistake is:
       "Pick X from the shelf."   then   "Place X on the shelf."     (INVALID)
       "Pick X from the basket."  then   "Place X in the basket."    (INVALID)
   These are hallucinations. The origin of a pick and the destination of the
   immediately-following place MUST be DIFFERENT. Rewrite them so they match
   the inferred material flow. Example, if the flow is basket -> shelf:
       "Pick a bottle from the basket with the right hand."
       "Place the bottle on the shelf with the right hand."

3. MERGE CONSECUTIVE DUPLICATES.
   - Combine consecutive lines whose action-object-location triple is
     identical into ONE line spanning the full duration.
   - If a line appears twice back-to-back with different timestamps, that is
     the same subtask continuing — merge.
   - "(none)" lines and empty entries: drop them.

4. REMOVE OBVIOUS HALLUCINATIONS.
   - Drop lines that describe an object clearly not present anywhere else in
     the timeline or the global context.
   - Drop lines that describe an impossible action given the surrounding
     context (e.g. picking from a surface that was just placed onto).

5. KEEP FINE GRANULARITY.
   - Do NOT aggregate many pick+place cycles into a single "restock" summary
     line. Each individual grasp and each individual release should remain a
     separate subtask, unless it is truly the SAME event continuing across
     adjacent frames.
   - Aim to preserve the roughly-1-second-per-action granularity of the
     original, but with correct semantics.

6. WORDING AND NAMING.
   - Use the SAME name for an object throughout (e.g. always "bottle" or
     always "shampoo bottle" — not both).
   - Prefer generic functional names ("bottle", "basket", "shelf", "mug").
   - REMOVE all color, size, shape, and material adjectives unless they
     disambiguate two objects of the same type in the same clip.
     WRONG: "blue shopping basket"  ->  RIGHT: "shopping basket"
     WRONG: "green bottle"         ->  RIGHT: "bottle"
   - Sentences: "Action + Object + Location/Context + Hand.", ending with a
     period.
   - EVERY manipulation line (pick, place, grasp, reach, extend arm) MUST
     specify the hand: "with the left hand", "with the right hand", or
     "with both hands". Never omit the hand on reach/extend lines.
   - Drop redundant "Grasp" immediately followed by "Pick" of the same object
     — keep "Pick" only.

7. TIMESTAMPS.
   - Preserve the FULL time coverage of the input window; output must span
     from the first input timestamp to the last in this window.
   - Do not invent new time ranges outside the observed [0, {duration_sec:.2f}] window.
   - Timestamps are continuous and non-overlapping in the output.
   - Format: [<start_sec>-<end_sec>] with 3 decimals.
