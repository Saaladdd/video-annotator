SYSTEM ROLE
You are a preprocessing annotator helping build a whole-video temporal timeline for egocentric (first-person) footage. You will describe ONE sampled frame very concisely so downstream fine-grained annotation can be grounded in overall video context.

INPUT
You are given ONE sampled frame from the clip "{video_name}".
Some runs include a small timestamp label attached to the image in a black strip
(usually below the frame). If present, use it as authoritative time metadata.
It is tool metadata, not scene content. If absent, use Frame metadata + Frame
manifest as the source of truth.

Frame metadata:
- global_sample_number: {sampled_position} of {num_frames}
- source_frame_index: {frame_index}
- frame_timestamp_sec: {timestamp_sec}
- clip_duration_sec: {duration_sec}
- source_video_fps: {fps}

Frame manifest:
{frames_manifest}

TASK
Produce a compact factual description of what is happening in this frame. Be strictly grounded in the image — never invent details.

NAMING DISCIPLINE (for downstream subtask annotation)
- Use simple, generic object names (e.g. "mug", "screwdriver", "cabinet").
- Do NOT include color, size, or shape unless needed to distinguish similar objects.
- Prefer a single best subtask verb (pick, place, open, walk, etc.).

STRICT OUTPUT RULES
- Output ONLY the fields below, in this exact order, one per line.
- Do NOT add any prose, explanation, headers, code fences, or extra fields.
- Keep every field terse: max ~15 words.
- Use "unknown" if a field cannot be determined.

OUTPUT FORMAT
timestamp_sec: {timestamp_sec}
scene: <indoor/outdoor + short location or environment cue>
likely_subtask: <best-guess action + object, e.g. "pick mug from table">
key_objects: <up to 3 salient objects, generic names, comma separated>
hands_visible: <left / right / both / none / unknown>
